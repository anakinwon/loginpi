# Frontend Design

이 스킬은 제네릭한 "AI 디자인"을 피하고, 의도적인 미학적 방향과 창의적인 선택이 담긴 프로덕션 수준의 프론트엔드 인터페이스 개발을 가이드합니다.

## 사용 방법

프론트엔드 요구사항을 제공하면:

1. **미학적 방향 결정** - 대담한 스타일 선택 (미니멀, 맥시멀, 레트로, 러그, 럭셔리, 장난스러운, 에디토리얼, 브루탈리스트 등)
2. **컨텍스트 분석** - 목적, 대상 사용자, 기술 제약
3. **차별화 포인트** - 사람들이 기억할 수 있는 독특한 특징
4. **프로덕션 구현** - 작동하는 코드 + 시각적으로 뛰어난 디자인

## 미학적 가이드라인

### Typography (타이포그래피)
- 아름답고 유니크한 폰트 선택 (Arial, Inter 같은 제너릭 피하기)
- 디스플레이 폰트 + 본문 폰트의 대조적 조합
- 글꼴이 전체 미학을 끌어올리도록

### Color & Theme (색상과 테마)
- 일관된 미학에 전념하기
- CSS 변수로 색상 관리
- 주도적 색상 + 날카로운 악센트

### Motion (애니메이션)
- CSS 애니메이션 우선 (HTML/Vanilla JS)
- React에서는 Motion 라이브러리
- 스크롤 트리거, 호버 상태, 페이지 로드 효과

### Spatial Composition (공간 구성)
- 예상 밖의 레이아웃
- 비대칭, 겹침, 대각선 흐름
- 넉넉한 음의 공간 또는 제어된 밀도

### Visual Details (시각적 세부사항)
- 그래디언트 메시, 노이즈 텍스처, 기하 패턴
- 맥시멀리스트 디자인 = 정교한 애니메이션과 효과
- 미니멀리스트 디자인 = 정밀하고 섬세한 디테일

## 피해야 할 것

❌ 제너릭 AI 미학
❌ 흔한 폰트 (Inter, Roboto, system fonts)
❌ 식상한 색상 조합 (특히 보라 그래디언트)
❌ 예측 가능한 레이아웃
❌ 맥락 없는 쿠키커터 디자인

## 핵심 원칙

- **의도성**: 대담한 미니멀이든 정교한 맥시멀이든, 명확한 개념적 방향과 정밀한 실행이 중요
- **맥락**: 각 디자인은 특정 목적, 사용자, 문제를 위해 유일해야 함
- **창의성**: 기술 제약 범위 내에서 가능한 한 창의적으로

## 빠른 시작

요구사항을 제공하면:

```
@frontend-design

[컴포넌트/페이지 설명]
- 목적: 무엇을 해결하는가?
- 톤/스타일: 어떤 미학적 방향?
- 제약: 기술적 요구사항?
```

이 스킬은 작동하는 코드와 비범한 디자인을 함께 제공합니다.
