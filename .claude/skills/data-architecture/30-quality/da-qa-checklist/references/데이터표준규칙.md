## 1. 테이블 명명 표준

### 1-1. 도메인 접두사 (필수)

모든 테이블은 업무 도메인을 나타내는 접두사로 시작해야 한다.

| 접두사 | 도메인 |
|--------|--------|
| `sys_` | 시스템 공통 (사용자, 권한, 설정) |
| `brd_` | 게시판/커뮤니티 (게시글, 댓글, 첨부) |
| `std_` | 표준 데이터 (단어, 도메인, 용어, 코드) |
| `pi_` | Pi Network 연동 (결제, 트랜잭션) |
| `auth_` | 인증/연동 (연동코드, 토큰) |
| `cod_` | 공통 코드 그룹/상세 |

**위반 사례 및 수정 이력**

| 변경 전 | 변경 후 | 이유 |
|---------|---------|------|
| `users` | `sys_user` | 시스템 사용자 — `sys_` 접두사 필요 |
| `payments` | `pi_pymnt` | Pi Network 결제 — `pi_` 접두사 필요 |
| `link_codes` | `auth_link_cd` | 인증 연동 코드 — `auth_` 접두사 필요 |
| `std_dic_sync` | `std_dic` | `_sync` 접미사는 비표준 |
| `std_dom_sync` | `std_dom` | 동일 |

### 1-2. 테이블명 형식
- 소문자 스네이크케이스 (`snake_case`)
- 복수형 금지 (`posts` → `brd_post`)
- 단어 수 3개 이하 권장
- 약어 사용 시 표준약어 목록 준수 (`pymnt`, `attch`, `cmnt`, `dtm`, `usr`)

## 2. 컬럼 명명 표준

### 2-1. 시스템 컬럼 (모든 테이블 필수, 반드시 테이블 마지막 4개 컬럼)

```sql
regr_id TEXT        NOT NULL DEFAULT 'ADMIN',              -- 등록자 ID
reg_dtm TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,    -- 등록일시
modr_id TEXT        NOT NULL DEFAULT 'ADMIN',              -- 변경자 ID
mod_dtm TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP     -- 변경일시 (트리거 자동 갱신)
```

**강제 규칙**:
- 4개 컬럼 모두 `NOT NULL` — nullable 금지
- `regr_id` / `modr_id` DEFAULT `'ADMIN'` — 시스템 작업 추적 기본값
- `reg_dtm` / `mod_dtm` DEFAULT `CURRENT_TIMESTAMP` — `NOW()` 동의어이나 표준 표기 준수
- 컬럼 순서: `regr_id → reg_dtm → modr_id → mod_dtm` (이 순서 고정)

**금지 패턴**: `created_at`, `updated_at`, `createdAt`, `updatedAt`, `create_dt`, `update_dt`

### 2-2. PK 컬럼
- UUID 타입 권장: `gen_random_uuid()` DEFAULT
- 컬럼명 패턴: `<엔터티약어>_id` (예: `dic_id`, `dom_id`, `term_id`, `post_id`)

### 2-3. FK 컬럼
- 참조 테이블의 PK 컬럼명을 그대로 사용
- 예: `sys_user.id` 참조 시 FK 컬럼명 = `user_id` 또는 `<역할>_usr_id`

### 2-4. Y/N 플래그 컬럼
```sql
<컬럼명>_yn CHAR(1) NOT NULL DEFAULT 'N' CHECK (<컬럼명>_yn IN ('Y', 'N'))
```
- 반드시 `CHAR(1)` — `TEXT`, `VARCHAR`, `BOOLEAN` 금지
- 주요 플래그: `del_yn` (논리 삭제), `use_yn` (사용 여부), `apv_yn` (승인 여부)

### 2-5. 수치 컬럼 타입 기준

| 범위 | 타입 |
|------|------|
| 카운트/일반 정수 (< 2억) | `INT` |
| 파일 크기, 금액, 큰 수 | `BIGINT` |
| 소수점 없는 금액 | `BIGINT` |
| 소수점 있는 금액 | `NUMERIC(precision, scale)` |

## 3. 논리 삭제 정책

### 3-1. 원칙
- 운영 데이터 테이블은 물리 `DELETE` 금지
- 모든 운영 테이블에 `del_yn CHAR(1) NOT NULL DEFAULT 'N'` 포함
- 삭제 시: `UPDATE SET del_yn='Y', mod_dtm=NOW(), modr_id=<현재사용자>`

### 3-2. 조회 시 필수 필터
```typescript
.from('std_dic').select('...').eq('del_yn', 'N')
```

### 3-3. 예외
- 순수 로그/이력 테이블 (append-only): 물리 삭제 허용
- 임시 데이터 테이블: 물리 삭제 허용

## 5. 트리거/함수 명명 표준

```sql
-- mod_dtm 자동 갱신 함수
fn_upd_<테이블명>_mod_dtm()

-- mod_dtm 자동 갱신 트리거  
trg_<테이블명>_mod_dtm  BEFORE UPDATE ON <테이블명>
```

각 운영 테이블은 UPDATE 시 `mod_dtm`을 자동 갱신하는 트리거를 보유해야 한다.
