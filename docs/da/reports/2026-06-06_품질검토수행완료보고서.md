## 1. 검토 개요

### 1-1. 검토 목적

프로젝트 Supabase 데이터베이스의 테이블·컬럼·제약조건 명명이 한국 DA 표준을 준수하는지 점검하고, 위반 사항을 일괄 수정하여 데이터 품질 준수율을 100%로 향상시키는 것을 목적으로 한다.

### 1-2. 검토 범위

| 구분 | 대상 |
|------|------|
| **데이터베이스** | Supabase PostgreSQL (프로덕션) |
| **테이블 수** | 10개 (`sys_user`, `pi_pymnt`, `auth_link_cd`, `brd_post`, `brd_cmnt`, `brd_attch`, `std_dic`, `std_dom`, `std_term`, `dept_info`) |
| **점검 항목** | 테이블명, 컬럼명, 시스템 컬럼, 논리 삭제, 제약조건, 트리거/함수 |
| **연관 코드** | Next.js 16 App Router API Route 18개 파일 |

### 1-3. 검토 기준 (적용 표준)

- 한국 DA 표준 — 도메인 접두사 기반 테이블 명명 (`sys_`, `brd_`, `std_`, `pi_`, `auth_`)
- 시스템 컬럼 suffix: `reg_dtm / mod_dtm / regr_id / modr_id`
- 논리 삭제 정책: `del_yn CHAR(1) DEFAULT 'N'`
- 참고: `품질점검기준서.pdf`가 공란이었으므로 프로젝트 관찰 패턴 + 업계 표준 적용

## 8. 향후 거버넌스 계획

### 8-1. 신규 테이블 생성 원칙

1. **도메인 접두사 필수**: `sys_`, `brd_`, `std_`, `pi_`, `auth_`, `cod_` 중 하나 선택
2. **시스템 컬럼 템플릿 의무 적용**:
   ```sql
   reg_dtm    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
   mod_dtm    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
   regr_id TEXT,
   modr_id TEXT
   ```
3. **운영 데이터 테이블**: `del_yn CHAR(1) NOT NULL DEFAULT 'N'` 필수
4. **`mod_dtm` 트리거**: 모든 UPDATE 대상 테이블에 자동갱신 트리거 생성

### 8-2. 코드 리뷰 게이트
