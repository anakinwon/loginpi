## 1. 위경도 표준 위반 (정정 완료)

### 1-1. 위반 내역 (정정 전)

| 테이블 | 컬럼 | 타입 | 위반 |
|--------|------|------|------|
| `mps_shop` | `lat`/`lng` | NUMERIC(9,6)/(10,6) | 표준단어·도메인·용어 3대 미준수 |
| `usr_loc_hist` | `lat`/`lng` | DECIMAL(10,8)/(11,8) | 동일 + 타입 불일치 |
| `mps_item` | `lat`/`lng` | DECIMAL(10,8)/(11,8) | 동일 (ALTER ADD COLUMN — Hook 사각지대 통과) |

- §1-3 위반: `lat`/`lng`가 표준도메인으로 끝나지 않음
- §1-3 위반: 위도/경도가 `std_dic` 미등록
- §1-2 위반: 좌표 도메인 `std_dom` 미정의 → 정밀도 100배 불일치(6자리 vs 8자리)

### 1-2. 정정 조치 (§0 Top-down 절차 준수)

| 단계 | 산출물 |
|------|--------|
| 표준단어 등재 | `std_dic`: 위도=`LATD`, 경도=`LNGT`, 좌표값=`CRD` |
| 표준도메인 등재 | `std_dom`: `CRD` = NUMERIC(11,8) |
| 표준용어 등재 | `std_term`: `latd_crd`, `lngt_crd` (도메인 CRD로 종료) |
| 물리모델 적용 | `sql/037_coord_standardize.sql` — 3테이블 rename + NUMERIC(11,8) 통일 + 인덱스 재생성 |
| 코드 반영 | `mps-shop.ts`·`mps-item.ts`·`location/save`·`nearby/shops`·`nearby/rooms` — DB는 표준명, JS/API는 별칭 매핑으로 `lat`/`lng` 유지 |

검증: `information_schema` 6개 컬럼 모두 `latd_crd`/`lngt_crd` `numeric(11,8)` 확인, `tsc` 0 errors.

## 2. 컬럼 명명 전수조사 (표준도메인 미종결)

전체 BASE TABLE 컬럼을 표준도메인 종결 기준으로 스캔. 결과를 **운영 위반**과 **레거시(미사용)** 으로 분류.

### 2-1. 운영 테이블 위반 (조치 필요 — 로드맵)

| 테이블 | 위반 컬럼(예) | 권장 표준용어 | 우선 |
|--------|--------------|--------------|------|
| `approval_queue` | decided_at, decided_by, req_at, req_by, reject_reason | `dcsn_dtm`, `dcsn_id`, `req_dtm`, `req_id`, `rjct_desc` | P2 |
| `auth_link_cd` | attempt_count, code, expires_at, used_at | `attempt_cnt`, `link_cd`, `exp_dtm`, `use_dtm` | P2 |
| `pi_pymnt` | amount, status, memo | `pymnt_amt`, `pymnt_sts`, `memo_cont` | P2 (코드 광범위) |
| `sys_user` | display_name, google_email, pi_username, self_intro | `disp_nm`, `google_eml`, `pi_user_nm`, `self_intro_cont` | P2 (코드 광범위) |
| `sys_user_consent` | user_agent | `user_agnt_cont` | P3 (LBS 신규 — 후속 정정) |
| `i18n_locale/message` | is_active, is_auto, msg_val | `use_yn`, `auto_yn`, `msg_cont` | P3 |
| `mps_order` | cancel_reason | `ccl_desc` | P3 |
| `mps_shop` | contact_email | `cntct_eml` | P3 |

> **단위 접미사 컬럼**(`accuracy_m`, `duration_sec`, `jitter_ms`, `rtt_ms`)은 측정단위로, 표준도메인 체계에 단위 도메인이 없음 → 별도 정책 결정 필요(P3).

### 2-2. 스타터킷 레거시 (미사용 — drop 대상)

`events`, `participants`, `profiles`, `role_logs`, `settlements`, `instruments`, `grp_mst`, `perm_mst`, `role_mst`, `prod_info`, `user_info`, `sync_log` — `created_at`·`name`·`email`·`title` 등 영어 자연어 명명. CLAUDE.md 현재 테이블 목록에 없는 초기 스타터킷 잔재. **개별 정정보다 일괄 논리삭제/drop 권장**.

### 핵심 교훈

`DA-APPROVED` 주석은 Hook이 **존재 여부만** 검사하고 사유 타당성은 판단하지 못한다. `036_item_location.sql`은 "위반 선례(029·033) 준수"라는 **순환 논리**로 자가 승인되어 통과했다. → `DA-APPROVED`를 다는 주체가 사유를 스스로 엄격히 검증해야 하며, 이 한계는 **정기 전수조사**(분기 1회, 본 skill의 SQL)로만 보완 가능하다.
