## 1. 프로젝트 궁극적 목적: 플랫폼 인큐베이팅

### 1.1 AI인큐베이터란?

**AI인큐베이터**는 Pi Network 생태계를 기반으로 한 **스타터킷 제품화** 플랫폼입니다.

고객의 니즈에 맞는:
- **전문 자문**: 플랫폼 아키텍처·기술 스택·라이선싱 전략 수립
- **구축 컨설팅**: 기능 커스터마이징·통합·배포·운영 노하우 제공
- **시간·비용 절약**: 검증된 패키지 모델로 가용성 극대화

### 1.2 포지셔닝

**대상**: Pi Network 생태계 파트너 (스타트업·중소 사업자·기업)

**핵심 제약 극복**: 
- Pi Browser WebView 쿠키 미저장 → X-Pi-Token 이중 경로 자동 처리
- 203개 locale 글로벌 배포 → 다국어·통화 자동 매핑
- Pi Coin 결제 → U2A 3단계 완전 구현 (미완료 결제 자동 복구)

**차별화**:
- **실시간 글로벌 동시통역** (PiTranslateâ¢™ — Gemini Flash + Claude Haiku 하이브리드)
- **P2P 직거래 에스크로** (PiShopâ¢ MPS — Pi Coin 자동 정산)
- **N:N 음성채널** (PiVoice™ — WebRTC Full Mesh + TURN)
- **위치기반 커머스** (LBS — 거리 기반 직거래 성사율 향상)
- **검증된 데이터 표준** (DA 시스템 — 표준단어·도메인·용어·DDL 자동 감사)

## 2. 시스템 구축 기본 요소: 4대 기술 축

### 2.1 AA 기술 (Application Architecture)

**목표**: 확장 가능하고 관리 용이한 풀스택 웹 앱 아키텍처

| 항목 | 선택 사항 | 설명 |
|---|---|---|
| 프레임워크 | **Next.js 16 App Router** | 최신 안정화, Turbopack 고속 빌드 |
| 런타임 | **React 19** | 최신 안정화 버전 |
| 타입 안전 | **TypeScript 6** | strict mode 강제 |
| 스타일 | **Tailwind CSS v4** | CSS-first, 번들 최소화 |
| UI 컴포넌트 | **shadcn/ui (@base-ui/react)** | 무헤드 컴포넌트, Pi Browser 호환 |
| 다국어 | **next-intl v4** | 203개 locale, 빌드 시점 검증 |
| 패키지 관리 | **pnpm 11** | 모노레포·monorepo, 고속 설치 |

**특징**:
- **Pi Browser + 일반 브라우저 동시 지원** — 환경 감지 조건부 렌더링
- **빌드 시점 환경변수 검증** (t3-env + Zod) → 배포 오류 조기 발견
- **무한 스크롤·지연 로딩** — 성능 최적화 (IntersectionObserver)

### 2.2 DA 기술 (Data Architecture)

**목표**: 기업 수준의 데이터 거버넌스와 품질 자동 관리

| 항목 | 구현 | 설명 |
|---|---|---|
| 데이터베이스 | **Supabase PostgreSQL** | 오픈소스, 실시간 broadcast, 글로벌 배포 |
| 접근 제어 | **서버 전용 service_role** | RLS 비활성화, 모든 접근 인증 서버 경로 |
| 표준 단어 | **표준 단어 사전** | 약어(REGR, PYMNT, CTGR 등) 중앙 관리 |
| 표준 도메인 | **도메인 집합** | 논리적 데이터 분류 (사용자, 결제, 게시판 등) |
| 표준 용어 | **논리명↔물리명 매핑** | 테이블·컬럼 명명 규칙 자동 검증 |
| DDL 감사 | **da-ddl-guard Hook** | `sql/*.sql` 작성 시 자동 검사, 위반 차단 |
| 시스템 컬럼 | **4대 필수 컬럼** | `regr_id`, `reg_dtm`, `modr_id`, `mod_dtm` |
| 논리삭제 | **del_yn + del_dtm** | 물리 DELETE 금지, 모든 삭제는 논리 마킹 |

**정본**: `docs/da/데이터표준규칙.md`

### 2.3 SaaS 기술 (Software-as-a-Service 통합)

**목표**: 타사 서비스 API 안전·효율적 통합

|---|---|---|

- **쿠키** (일반 브라우저, Pi Browser 미지원)

## 5. 제약 및 핵심 고려 사항

### 5.1 Pi Browser 제약 극복

**문제**: Pi Browser WebView는 모든 방식의 `Set-Cookie` 미저장

**해결책**:
- **쿠키 (일반 브라우저용)**
- **`X-Pi-Token` 헤더** (Pi Browser localStorage 기반)
- **클라이언트 게이트**: `getSessionUser()` null 시 redirect 금지 (무한 루프 방지)
