# MVP PRD 생성 메타 프롬프트

> 이 파일은 Claude Code에 붙여넣어 MVP PRD 문서를 자동 생성하는 **메타 프롬프트**입니다.
> 산출물은 `docs/PRD.md`로 저장됩니다.

---

## 사용법

아래 구분선(`---`) 안의 내용 전체를 Claude Code 채팅창에 붙여넣으세요.

---

```
당신은 1인 개발자를 위한 MVP PRD(Product Requirements Document) 전문 작성가입니다.
현재 프로젝트의 코드베이스를 분석하고, 아래 절차에 따라 `docs/PRD.md`를 생성해 주세요.

## 역할 정의

- 역할: 시니어 프로덕트 매니저 + 테크 리드
- 목표: 실제 개발에 바로 착수할 수 있는 구체적이고 간결한 MVP PRD 작성
- 원칙: 과도한 기획보다 **지금 당장 만들 수 있는 것**에 집중

---

## STEP 1 — 코드베이스 자동 분석

다음 파일들을 순서대로 읽어 프로젝트 컨텍스트를 파악하세요.

1. `CLAUDE.md` — 기술 스택, 명령어, 아키텍처 규칙
2. `AGENTS.md` — 에이전트 협업 구조 (있다면)
3. `package.json` — 의존성 및 스크립트
4. `app/` 또는 `src/` — 현재 구현된 페이지/기능
5. `docs/ROADMAP.md` — 이미 수립된 로드맵 (있다면)
6. `README.md` — 프로젝트 소개 (있다면)

분석 완료 후 다음을 파악하세요:
- 현재 구현 완료된 기능 목록
- 사용 중인 핵심 라이브러리
- 폴더 구조 패턴
- 데이터 흐름 방식 (API Routes / Server Actions / External API)

---

## STEP 2 — 사용자에게 보완 정보 수집

코드베이스 분석 후, 아래 질문을 **한 번에 모두** 제시하세요.
(사용자가 이미 답한 내용은 건너뛰세요)

### 필수 질문 (반드시 확인)

**[Q1] 이 제품은 누구를 위한 것인가요?**
예시: "소규모 쇼핑몰 운영자", "프리랜서 디자이너", "스타트업 HR 담당자"

**[Q2] 해결하려는 핵심 문제는 무엇인가요?**
현재 사용자가 겪는 불편함, 비효율, 또는 공백을 한 문장으로 설명해 주세요.

**[Q3] MVP에 반드시 있어야 할 기능 3가지는?**
우선순위 순으로 나열해 주세요. (더 많아도 되지만 MVP는 3개로 시작)

**[Q4] MVP 목표 완성 기간은?**
예시: "2주", "1개월", "3개월"

**[Q5] 성공의 기준은 무엇인가요?**
숫자로 측정 가능한 지표를 제시해 주세요.
예시: "MAU 100명", "전환율 5%", "하루 처리 건수 50건"

### 선택 질문 (해당하는 경우만)

**[Q6] 경쟁 제품 또는 참고할 서비스가 있나요?**

**[Q7] 수익 모델이 있나요?** (SaaS 구독, 1회 결제, 광고, 무료 등)

**[Q8] 외부 연동이 필요한 서비스가 있나요?**
예시: "카카오 로그인", "토스 결제", "Slack 알림"

## STEP 4 — 완료 후 체크리스트

PRD 작성 완료 후 다음을 수행하세요:

1. `docs/PRD.md` 파일 저장 확인
2. `CLAUDE.md`에 PRD 경로 참조 추가 (없다면):
   ```markdown
   - **📋 프로젝트 요구사항**: `@/docs/PRD.md`
   ```
3. 사용자에게 다음 메시지 출력:

```
✅ docs/PRD.md 생성 완료

다음 단계를 추천합니다:
1. PRD 내용 검토 후 누락된 요구사항 추가
2. `/docs:update-roadmap` 실행 → ROADMAP.md와 PRD 동기화
3. `git add docs/PRD.md && git commit -m "docs: MVP PRD 초안 작성"`
```
```

## 참고: PRD 품질 기준

| 기준 | 좋은 PRD | 나쁜 PRD |
|------|---------|---------|
| 기능 설명 | "로그인 실패 3회 시 계정 잠금 10분" | "보안 강화" |
| 성공 지표 | "DAU 50명, 이탈율 30% 이하" | "많은 사용자" |
| 범위 | 포함/제외 명확히 구분 | 모호한 경계 |
| 공수 | 기능별 예상 시간 명시 | 일정 없음 |
| 우선순위 | P0/P1/P2 구분 | 모든 기능이 중요 |

---

> **팁**: PRD는 완벽할 필요 없습니다. 80% 완성도로 빠르게 작성하고 개발하면서 업데이트하세요.
> 이 메타 프롬프트 자체도 프로젝트에 맞게 수정해서 사용하세요.
