## 1. 개요 (Abstract)

**BEAN**은 Pi Network 메인넷 기반 커뮤니티·커머스 플랫폼 **Cafe.pi**의 생태계 **유틸리티 토큰**이다. BEAN은 자금 조달 수단이 아니라, Cafe.pi 안에서 **사용자 간 후원·보상·기능 사용권**을 매개하는 도구다.

BEAN은 이미 Cafe.pi에서 운영 중인 인앱 후원 단위 **"Pi Bean"(🫘)** 을 온체인 토큰으로 확장한 것으로, 실제 작동하는 제품 위에서 즉시 효용을 갖는다. 발행은 Pi Network의 공식 **Launchpad**를 통해 이루어지며, Launchpad의 **Product-First(제품 우선)** 원칙 — 토큰은 자본 조달이 아닌 사용자 획득·제품 효용을 위한 것 — 과 정합한다.

| 핵심 | 값 |
|---|---|
| 토큰명 | BEAN |
| 발행 체인 | Pi Network (Stellar 기반) · Soroban 스마트 컨트랙트 |
| 총 발행량 | 1,000,000,000 BEAN (10억, 고정) |
| 기축 | Pi (BEAN/Pi 단독) |
| 성격 | 유틸리티 토큰 (사용권·보상·후원) |

---

## 2. 배경 — Cafe.pi 생태계

Cafe.pi는 Pi Network 메인넷 위에서 동작하는 **온라인 커뮤니티 + O2O 마켓플레이스**다.

| 구성 | 설명 |
|---|---|
| **PyChat (카페)** | 테마 기반 실시간 커뮤니티. 같은 관심사의 Pi 사용자들이 모이는 공간 |
| **PyShop™ (마켓플레이스)** | Pi 결제 기반 P2P 직거래 + O2O 오프라인 매장 커머스 |
| **PyVoice (음성채팅)** | WebRTC 기반 N:N 음성 채널 |
| **이벤트·미션** | 사용자 활동 보상 및 커뮤니티 활성화 |

Cafe.pi에는 이미 **Pi Bean 후원 기능**이 존재한다 — 사용자가 좋은 콘텐츠·도움을 준 상대에게 소액을 후원하는 문화다. 그러나 현재 Bean은 단순 인앱 표시 단위에 머물러, **온체인 자산으로서의 이전성·상호운용성·투명성**을 갖지 못한다.

**BEAN 토큰은 이 검증된 후원 문화를 온체인으로 끌어올려**, 후원·보상·기능 사용권을 하나의 표준 단위로 통합한다.

---

## 3. BEAN 토큰 정의 (유틸리티)

BEAN의 **1차 목적은 유틸리티**다 — Cafe.pi 생태계 내에서의 사용권과 보상이며, 투자 수익을 위한 것이 아니다.

- ✅ **사용권**: 카페·마켓·음성 기능 이용 및 할인 · **글로벌 동시통역(PyTranslate™) 이용** · **마켓 구매/판매 수수료 결제**
- ✅ **후원**: 사용자 간 Bean 후원(기존 Pi Bean의 온체인화) · **도네이션 기반 사회 기부 연계**
- ✅ **보상**: 미션·이벤트·커뮤니티 기여 보상 · **이용후기(리뷰) 보상** · **매출 쿠폰/스탬프 보상**
- ✅ **(향후) 거버넌스**: 생태계 정책 투표 참여권 (수익권과 분리)

> BEAN은 **배당·이자·수익 분배를 제공하지 않으며**, 발행 주체는 BEAN의 시장 가격 상승을 약속하거나 그를 위해 노력할 것을 약정하지 않는다. (§10 참조)

### 기존 Pi Bean과의 관계 (단위 정합)

온체인 전환에 따라 인앱 후원 단위를 다음과 같이 정렬한다:

```
1 Pi = 100 BEAN
기존 후원 0.1 / 0.5 / 1 Pi  →  10 / 50 / 100 BEAN
```

이 정합은 사용자가 이미 익숙한 후원 경험을 그대로 유지하면서 단위를 정수로 단순화한다.

## 4. 사용처 (Utility Use Cases)

| # | 사용처 | 설명 |
|---|---|---|
| 1 | **후원(Tip)** | 카페·콘텐츠 기여자에게 BEAN 후원 (기존 Pi Bean 온체인화) |
| 2 | **사회 기부(Donation)** | BEAN 도네이션을 모금·기부 캠페인으로 연계 — 커뮤니티의 사회적 기여 채널 |
| 3 | **기능 사용권** | 프리미엄 테마·스티커·AI 비서 등 기능 잠금 해제 |
| 4 | **글로벌 동시통역(PyTranslate™)** | 실시간 번역 기능 이용권 결제 |
| 5 | **마켓 거래 수수료** | PyShop™ 구매/판매 수수료를 BEAN으로 결제 |
| 6 | **미션·이벤트 보상** | 활동 보상으로 BEAN 지급 (생태계 리저브 재원) |
| 7 | **이용후기(리뷰) 보상** | 카페·매장·상품 리뷰 작성 기여에 BEAN 보상 |
| 8 | **매출 쿠폰·스탬프 보상** | O2O 오프라인 매장 구매 적립(스탬프)·쿠폰을 BEAN으로 환원 |
| 9 | **(향후) 거버넌스** | 생태계 정책·로드맵 투표 |

> 모든 사용처는 **소비·사용 중심**이며, 보유에 따른 금전적 수익을 전제하지 않는다.

## 5. 토크노믹스

### 5.1 기본 파라미터

| 항목 | 값 |
|---|---|
| 총 발행량 | 1,000,000,000 BEAN (고정, 추가 발행 없음) |
| 소수점 | 7자리 (1 BEAN = 10,000,000 units, i128) |
| 기축 | Pi |
| 참여 단가(세일) | 0.01 Pi / BEAN |

### 5.2 분배 구조

| 카테고리 | 비율 | 수량 | 락업/베스팅 | 목적 |
|---|---|---|---|---|
| **Launchpad 세일** | 40% | 400,000,000 | 즉시~짧은 cliff | 커뮤니티 분배·참여 |
| **생태계 리저브** | 25% | 250,000,000 | 3~4년 분할 방출 | 사용자 보상·미션 재원 |
| **유동성 풀** | 15% | 150,000,000 | 영구 락 | BEAN/Pi 유동성 |
| **마케팅/파트너십** | 12% | 120,000,000 | 12개월 락 | 생태계 협력·커뮤니티 |
| **팀/재단** | 8% | 80,000,000 | 6개월 cliff + 36개월 선형 | 장기 운영 정렬 |

> **설계 원칙**: 인사이더(팀) 비중을 8%로 최소화하고 생태계·커뮤니티 배분(세일+리저브 65%)을 두텁게 하여, 토큰의 **효용 중심 분배**를 명확히 한다.

### 5.3 팀 베스팅

| 구간 | 기간 | 해제 |
|---|---|---|
| Cliff | 0~6개월 | 0% |
| Year 1 | 6~12개월 | 25% (분기 6.25%) |
| Year 2 | 1~2년 | 50% (분기 6.25%) |
| Year 3 | 2~3년 | 25% (분기 6.25%) |

## 6. 분배 및 Launchpad 참여

BEAN은 Pi Network 공식 **Launchpad**를 통해 분배된다.

- **Fair-Access(공정 접근)**: 참여자는 약정액(1~500 Pi)을 선택하고, Launchpad가 약정에 따라 보유 비율을 자동 산정하여 **소수 대형 참여자의 독점을 방지**한다.
- **유동성 예치**: 참여로 모인 Pi는 **발행 주체에게 직접 귀속되지 않고 유동성 풀에 예치**된다 — Launchpad 구조의 핵심으로, 토큰이 자금 조달이 아닌 생태계 유동성·효용을 위한 것임을 구조적으로 보장한다.
- **유동성 페어**: BEAN/Pi **단독**. (Pi 외 암호화폐·법정화폐 연동 페어는 운용하지 않는다.)

> 참여 단가(0.01 Pi/BEAN)는 **생태계 참여 파라미터**이며, 향후 시장 가격이나 수익을 시사하지 않는다.

## 7. 기술 아키텍처

- **체인**: Pi Network (Stellar Consensus Protocol 기반)
- **컨트랙트**: Soroban 스마트 컨트랙트 (Cafe.pi의 PiRC2 구독 컨트랙트와 동일 환경)
- **단위계**: 1 BEAN = 10,000,000 units (i128) — Pi 단위계와 정렬
- **오버플로우 안전성**: 총량 10억 BEAN = 10¹⁶ units ≪ i128 최대값(약 1.7×10³⁸) → 안전
- **표준 기능**: 발행(mint, 고정 총량)·전송(transfer)·승인(approve/transferFrom)·잔액 조회

배포는 Testnet 개발·테스트 → PiRC 커뮤니티 공개 검토 → 외부 보안 감사 → Mainnet 배포 순으로 진행한다.

## 8. 거버넌스 (향후)

초기 BEAN은 사용권·후원 중심이며, 거버넌스는 생태계 성숙 후 단계적으로 도입한다. 거버넌스 투표권은 **수익권과 명확히 분리**되며, 보유에 따른 금전적 권리를 부여하지 않는다.

## 10. 리스크 및 법적 고지

> ⚠️ 본 섹션은 **법무 자문(§8-1-1) 회신 후 최종 확정** 예정 — 초안입니다.

- **비투자·비증권 고지**: BEAN은 **유틸리티 토큰**이며 투자 상품이 아니다. 본 백서는 투자 권유·청약 권유가 아니다.
- **수익 미보장**: 발행 주체는 BEAN의 가치·가격을 보장하지 않으며, 배당·이자·수익 분배를 제공하지 않는다.
- **가치 변동**: 토큰의 가치는 변동할 수 있으며, 참여자는 본인 판단과 책임으로 참여한다.
- **규제 리스크**: 토큰 관련 규제는 관할별로 상이하며 변경될 수 있다. 일부 관할에서는 참여가 제한될 수 있다.
- **기술 리스크**: 스마트 컨트랙트는 보안 감사를 거치나 기술적 위험을 완전히 배제할 수 없다.
- **계획의 변경 가능성**: 본 백서의 내용(토크노믹스·로드맵 포함)은 규제·기술·생태계 상황에 따라 변경될 수 있다.
