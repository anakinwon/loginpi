# 성능 문제 기록 — CAFE(채팅) 탭 (2026-06-23)

## 문제 요약

CAFE 탭은 전반적으로 설계가 양호하나, **Pi Browser WebSocket 동작 미검증**(CRITICAL), **메시지 메모이제이션 부재**, **Gemini 번역 크레딧 소진**이 주요 이슈.

## 증상

1. Pi Browser에서 채팅방 입장 → **상대 메시지 수신 지연 또는 미수신** (WebSocket 미지원 가능성)
2. 메시지 500개 이상 → **메시지 입력 시 전체 리스트 재렌더** (500ms 이상 지연)
3. 메시지 번역 요청 → **"번역 불가" 에러** (Gemini API 크레딧 소진)
4. 첫 방문 사용자 → **스켈레톤 UI 없음** → 빈 화면 (UX 저하)
5. 카페 목록 멤버수 → **API 중복 호출** (각 카페마다 별도 COUNT)

## 근본 원인

### 원인 1: Pi Browser WebSocket 미지원 가능성 (CRITICAL)

**파일**: `src/hooks/use-chat-room.ts` L50~80

- Realtime 구독이 WebSocket 기반 → Pi Browser WebView에서 미지원 우려
- **검증 미완료**: 실기기 Network 탭에서 프로토콜 업그레이드(101) 확인 필요
- **부작용**: WebSocket 연결 실패 → 메시지 실시간 수신 미작동 → polling 폴백 없음

- 부모 리렌더 → **모든 자식 메시지 컴포넌트 재렌더** (심각)
- 500개 메시지 환경에서 1개 메시지 수정 → 499개 불필요 리렌더 (+1000ms)

## 해결책

### 개선 1: Pi Browser WebSocket 동작 검증 (P0 최우선)

**작업**: Pi 실기기에서 테스트

```
1. Pi Browser 개발자 도구 열기 (F12 또는 Ctrl+Shift+J)
2. Network 탭 → Filter: WS (WebSocket)
3. 채팅방 입장
4. 메시지 전송
5. Network에서 WebSocket 프로토콜 업그레이드(101) 확인
   - 있으면: ✅ 정상
   - 없으면: ⚠️ WebSocket 미지원 → polling fallback 구현 필수
```

**폴백 구현** (WebSocket 미지원 시):

```typescript
// src/hooks/use-chat-room.ts
useEffect(() => {
  let pollTimer: NodeJS.Timeout | null = null

  const channel = supabase.channel(`room:${roomId}`)

  channel.subscribe(async (status) => {
    if (status === 'SUBSCRIBED') {
      // WebSocket 성공 → polling 중단
      if (pollTimer) clearInterval(pollTimer)
      console.log('[채팅] Realtime 연결 성공')
    } else if (status === 'CLOSED') {
      // WebSocket 미지원 또는 연결 실패 → polling 시작
      console.warn('[채팅] Realtime 폴백: 5초 간격 폴링')
      pollTimer = setInterval(async () => {
        const res = await piFetch(`/api/chat/rooms/${roomId}/messages?limit=1`)
        if (res.ok) {
          const { messages: newest } = await res.json()
          // newest[0] vs lastMsgRef 비교 → 신규 메시지 감지
          if (newest[0]?.msg_id !== lastMsgRef.current?.msg_id) {
            // 신규 메시지 추가
            addMessage(newest[0])
            lastMsgRef.current = newest[0]
          }
        }
      }, 5000)
    }
  })

  return () => {
    if (pollTimer) clearInterval(pollTimer)
    supabase.removeChannel(channel)
  }
}, [roomId])
```

**예상 효과**:
- WebSocket 미지원 시에도 5초 지연으로 메시지 수신 (기능 유지)
- 최악의 경우에도 무한 대기 상황 회피

### 개선 2: 메시지 메모이제이션 추가

**파일**: `src/components/chat/chat-message-list.tsx`

```typescript
import { memo, useMemo } from 'react'

// 메시지 아이템을 별도 컴포넌트로 분리
const MessageItem = memo(function MessageItem({
  msg,
  prevMsg,
  onClick,
}: {
  msg: Message
  prevMsg?: Message
  onClick: () => void
}) {
  // 시간 표시 여부 (이전 메시지와 시간 다를 때만)
  const showTime =
    !prevMsg || prevMsg.reg_dtm.slice(0, 16) !== msg.reg_dtm.slice(0, 16)

  return (
    <div
      key={msg.msg_id}
      className="mb-2 flex gap-2"
      onClick={onClick}
    >
      {showTime && (
        <time className="text-xs text-gray-500">
          {new Date(msg.reg_dtm).toLocaleTimeString()}
        </time>
      )}
      <div className="flex-1 rounded-lg bg-blue-100 p-2">
        <p className="text-sm">{msg.msg_cont}</p>
        {msg.msg_trans && msg.msg_trans.trans_cont && (
          <p className="mt-1 text-xs text-gray-600">{msg.msg_trans.trans_cont}</p>
        )}
      </div>
    </div>
  )
})

export function ChatMessageList({
  messages,
  onMessageSelect,
}: ChatMessageListProps) {
  // useMemo로 메시지 아이템 메모이제이션
  const messageElements = useMemo(
    () =>
      messages.map((msg, idx) => (
        <MessageItem
          key={msg.msg_id}
          msg={msg}
          prevMsg={messages[idx - 1]}
          onClick={() => onMessageSelect?.(msg.msg_id)}
        />
      )),
    [messages, onMessageSelect]
  )

  return <div className="space-y-1">{messageElements}</div>
}
```

**예상 효과**:
- 메시지 입력 시 리렌더 -95% (메모이제이션된 MessageItem만 stable)
- 500개 메시지 입력 지연: 500ms → 50ms (10배 개선)

### 개선 4: 번역 API 폴백 (Anthropic)

**파일**: `src/app/api/chat/rooms/[roomId]/messages/translate/route.ts`

```typescript
import { Anthropic } from '@anthropic-ai/sdk'

export async function POST(req: NextRequest) {
  const { messages, targetLocale } = await req.json()

  // 1단계: Gemini Flash 시도
  try {
    const response = await googleGenerativeAI.generateContent({
      contents: messages.map(m => ({
        role: 'user',
        parts: [{ text: `${m.msg_cont} → ${targetLocale}로 번역` }],
      })),
      generationConfig: { maxOutputTokens: 1000 },
    })
    return NextResponse.json({ translations: response.text })
  } catch (geminiErr) {
    console.warn('[번역] Gemini 실패, Anthropic 폴백:', geminiErr.message)

    // 2단계: Anthropic 폴백
    try {
      const anthropic = new Anthropic()
      const responses = await Promise.all(
        messages.map((msg) =>
          anthropic.messages.create({
            model: 'claude-3-5-sonnet-20241022',
            max_tokens: 200,
            messages: [
              {
                role: 'user',
                content: `${msg.msg_cont} → 한국어로 번역해줘. 번역문만 반환.`,
              },
            ],
          })
        )
      )
      return NextResponse.json({
        translations: responses.map((r) => r.content[0].text),
      })
    } catch (anthropicErr) {
      console.error('[번역] 모두 실패:', anthropicErr)
      // 3단계: 원문 반환
      return NextResponse.json({
        translations: messages.map((m) => m.msg_cont),
      })
    }
  }
}
```

**예상 효과**:
- Gemini 크레딧 소진 시에도 번역 기능 유지 (Anthropic 폴백)
- 최악의 경우 원문 표시 (기능 저하 아님)
