# 성능 문제 기록 — SHOP(PiShop™) 탭 (2026-06-23)

## 문제 요약

SHOP 탭은 기능 완성도가 높으나, **StoreItemList의 과도한 상태 관리(13개 state)**와 **중복 API 호출 로직**이 주요 병목. 또한 **이미지 최적화 미흡**과 **Pi Browser GPS 권한 반복 요청**이 사용자 경험을 저하.

## 증상

1. 검색어 입력 → 300ms debounce 중 정렬 변경 → **API 2회 호출** (중복)
2. 상품 목록 100개+ → 필터 변경 시 **모든 카드 재렌더링** (메모이제이션 미흡)
3. 근처 상품 조회 → **GPS 권한 대화상자 반복 표시** (세션 캐시 부재)
4. 상품 이미지 → **원본 1280px 로드** (thumbnail 최적화 미흡)
5. Pi Browser 저사양 환경 → 상품 목록 초기 로드 **5초 이상**

## 근본 원인

### 원인 1: 클라이언트 상태 과다 관리

- Pi Browser 저사양 환경(메모리 128MB) → 초기 로드 +500ms

### 개선 3: GPS 권한 세션 캐싱

**파일**: `src/components/store/store-item-list.tsx` L100~121

```typescript
const requestLocation = useCallback((fresh = false) => {
  if (!fresh) {
    // sessionStorage에서 캐시 확인 (탭 세션 동안만)
    const cached = sessionStorage.getItem('shop_user_loc')
    if (cached) {
      const [lat, lng] = JSON.parse(cached)
      setUserLat(lat)
      setUserLng(lng)
      return
    }
  }

  getCurrentPosition({ fresh })
    .then(({ latitude, longitude }) => {
      setUserLat(latitude)
      setUserLng(longitude)
      // 캐시 저장 (탭 닫힐 때까지 유지)
      sessionStorage.setItem('shop_user_loc', JSON.stringify([latitude, longitude]))
    })
    .catch((e: Error) => setLocError(e.message))
}, [])

useEffect(() => {
  if (lbsConsent === 'Y' && !userLat) requestLocation()
}, [lbsConsent, userLat, requestLocation])
```

**예상 효과**:
- GPS 권한 대화상자 -50% (탭 재방문 시 0회)
- "주변순" 버튼 클릭 시 지연 -3초 (권한 요청 스킵)
