# 성능 문제 기록 — MAP 탭 (2026-06-23)

## 문제 요약

MAP 탭은 전반적으로 설계가 양호하나, **마커 클러스터링 부재**로 100개 이상 마커 시 지도 렉 발생, **마커 재렌더링 최소화 미흡**으로 shops 변경 시마다 모든 마커 재생성.

## 증상

1. Pi 등록 매장 100개 이상 → 지도 마커 클릭 응답 지연 (500ms 이상)
2. 사용자 위치 갱신(watchPosition) → shops 배열 재조회 → 기존 마커 제거 후 재생성 → 깜빡임
3. 같은 좌표에서 새로고침 → API는 동일한 shops 반환하지만 마커 모두 재렌더링

## 근본 원인

### 원인 1: 마커 클러스터링 미구현

**파일**: `src/components/lbs/shops-map-view.tsx` L120~200

- `@googlemaps/markerclusterer` 라이브러리 설치 안 됨
- 100+ 마커의 DOM 이벤트 리스너 개별 등록 → 메모리 누수 위험
- InfoWindow 1개 재사용하지만 HTML 재생성 반복 → 브라우저 reflow 빈번

### 원인 2: 마커 재렌더링 최소화 미흡

**파일**: `src/components/lbs/shops-map-view.tsx` - useEffect 의존성

- shops 변경될 때마다 `clearMarkers()` → `addMarker()` 반복
- 동일한 데이터(예: 30m 이동 후 재조회 = 동일 shops 배열)도 재렌더링
- 이전 shops와 현재 shops 비교 로직 없음

### 원인 3: Google Places API 중복 호출

**파일**: `src/components/lbs/shops-map-view.tsx` L150~170

- bizCategory != 'ALL' 시 Place.searchNearby() 별도 호출
- 그런데 nearby/shops API로 이미 Pi 매장을 조회함
- 결과: 카페 카테고리 선택 → Pi 매장 + Google 검색 결과 중복 표시

## 해결책

### 개선 1: 마커 클러스터링 추가 (P0 최우선)

**파일**: `src/components/lbs/shops-map-view.tsx`

```bash
pnpm add @googlemaps/markerclusterer
```

```typescript
import { MarkerClusterer } from '@googlemaps/markerclusterer'

useEffect(() => {
  if (!shops.length || !mapInstanceRef.current) return

  // 기존 마커 초기화
  clearMarkers()

  // 마커 생성
  const markers: google.maps.marker.AdvancedMarkerElement[] = []
  for (const shop of shops) {
    const marker = new AdvancedMarkerElement({
      map: mapInstanceRef.current,
      position: { lat: shop.lat, lng: shop.lng },
      title: shop.shop_nm,
    })
    markers.push(marker)
  }

  // 클러스터링 적용
  const clusterer = new MarkerClusterer({
    map: mapInstanceRef.current,
    markers,
  })

  markerMapRef.current = new Map(
    markers.map((m, idx) => [shops[idx].shop_id, m])
  )
}, [shops, bizCategory])
```

**예상 효과**:
- 100+ 마커 상황에서 UI 반응성 10배 향상
- 메모리 사용량 30% 감소 (클러스터 아이콘 재사용)
- 지도 클릭/줌 성능 250ms → 50ms

### 개선 2: 마커 재렌더링 최소화

**파일**: `src/components/lbs/shops-map-view.tsx`

```typescript
const [prevShopsRef, setPrevShopsRef] = useState<NearbyShop[]>([])

useEffect(() => {
  // 실제 변경만 감지
  const shopsChanged = 
    shops.length !== prevShopsRef.length ||
    JSON.stringify(shops) !== JSON.stringify(prevShopsRef)

  if (!shopsChanged && mapInstanceRef.current) return

  setPrevShopsRef(shops)

  // 마커 초기화 및 재생성
  clearMarkers()
  // ... addMarker 루프
}, [shops, prevShopsRef])
```

**예상 효과**:
- 동일 데이터 재조회 시 마커 깜빡임 제거
- 성능: 초기 5~10% 개선 (reflow 횟수 감소)
